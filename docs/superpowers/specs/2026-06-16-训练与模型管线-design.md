# 训练与模型管线设计（Phase 1）

> 智能排绳故障检测与数字孪生系统 —— 模型训练侧代码框架
> 日期：2026-06-16　状态：已确认设计，待评审 spec

## 1. 背景与范围

当前**尚无真实视频/图像数据**，本阶段只交付一套**可端到端运行的训练侧代码框架**；真实数据到位后替换数据集即可正式训练。

**本 spec 只覆盖 Phase 1（训练与模型管线）。** 边缘推理进程、FFmpeg 转发、MQTT、上位机（LabVIEW）属于 Phase 2，另起 spec。

**运行位置说明**：训练运行在带 GPU 的开发/训练机上；"代码要能跑在大部分工控机上"是 Phase 2 边缘**推理**侧的约束，本阶段通过"导出 onnx/openvino/engine 多后端 + 后端可切换"为其铺路。

## 2. 已确认的关键决策

| 决策点 | 结论 |
|---|---|
| 检测范式 | **图像分类**（把锁定 ROI 判成 正常/故障 → 0/1）。OBB 不做。 |
| 推理后端 | **后端可切换**：NVIDIA→TensorRT、Intel→OpenVINO、纯 CPU→ONNXRuntime |
| 类别 | **二分类**：`normal` / `fault` 两个文件夹（图像级标签） |
| 标注策略 | **直接按 `正常/故障` 文件夹分类标注**（无需画框，无 OBB） |
| 模型对比 | **YOLOv8s-cls 与 YOLOv8n-cls** 两个模型（small vs nano，服务"跑在大部分工控机上"）。不做 YOLO11、不做 nano 以外的额外变体。 |
| 分辨率 | **320 与 640 两种都试**（每模型 ×2，共 4 个 run），benchmark 里对比，兼顾"保螺旋纹理"与"速度" |
| 代码结构 | **A. 配置驱动的实验运行器**：`experiments.yaml` 列出"模型×分辨率"，脚本统一遍历 |

## 3. 目录与文件

```
configs/experiments.yaml      # 4 个 run：v8s-cls@320/@640、v8n-cls@320/@640（共享超参 + 可逐项覆盖）
tools/
  make_dummy_dataset.py       # 生成少量合成图直接落到 normal/ 与 fault/ 文件夹，验证管线
  copy_paste_aug.py           # 离线增强：故障小块随机贴到正常背景 → 存为 fault 类图，造故障样本
train.py                      # 配置驱动：依次训练 4 个 run（仅分类）
export.py                     # best.pt → onnx + openvino +（检测到 CUDA 才）engine/FP16
benchmark.py                  # 各 run×后端 实测 F1 / 单帧 ms / 显存(内存) MB → 输出对比表（md + csv）
```

数据集目录（分类按文件夹）：
```
dataset_cls/
  train/{normal,fault}/*.jpg
  val/{normal,fault}/*.jpg
```

## 4. 各组件职责（单一职责、可独立运行）

### 4.1 `configs/experiments.yaml`
- `common`：epochs、batch、device、project 等共享超参。
- `models`：列表，每项 `{name, weights, imgsz, 可选 epochs 覆盖}`。
- 默认 4 项：
  - `v8s-cls-320`（yolov8s-cls.pt, imgsz 320）
  - `v8s-cls-640`（yolov8s-cls.pt, imgsz 640）
  - `v8n-cls-320`（yolov8n-cls.pt, imgsz 320）
  - `v8n-cls-640`（yolov8n-cls.pt, imgsz 640）
- **device 自动**：可写 `auto`，运行时有 CUDA 用 GPU、否则回退 CPU。

### 4.2 `tools/make_dummy_dataset.py`
- 用 OpenCV/PIL 程序化生成数十张合成图：正常=平行斜纹，故障=叠加交叉线/隆起斑块。
- 直接写入 `dataset_cls/{train,val}/{normal,fault}/`，让无真实数据时也能跑通全链路。
- 额外生成几张"故障小块"到 `patches/` 供 Copy-Paste 演示。**仅用于验证，不进入正式训练。**

### 4.3 `tools/copy_paste_aug.py`（对应方案第八节）
- 输入：`patches/`（真实故障小块，现阶段可用 dummy）+ `dataset_cls/train/normal/`（正常背景）。
- 把故障小块随机位置（可小角度、轻微缩放）贴到正常背景，输出图存入 `dataset_cls/train/fault/`。
- **禁用大角度旋转**（保平行纹理朝向恒定，避免模型学偏）。
- 离线运行，扩充故障类样本。

### 4.4 `train.py`
- 读 `experiments.yaml`，对每个 run 调用 `YOLO(weights).train(task='classify', data='dataset_cls', imgsz=...)`。
- 在线增强：HSV/亮度/对比度/模糊/噪声开启以抗环境光；`degrees=0`（禁大角度旋转），保留 `translate`、轻微 `scale`、`fliplr`。
- 结果存 `runs/<name>/`。

### 4.5 `export.py`
- 对每个 run 的 `best.pt` 导出：`onnx`（通用 CPU）、`openvino`（Intel）、`engine`（TensorRT/FP16，仅在检测到 CUDA 时）。
- 缺少某后端依赖时跳过并告警，不中断其余导出。

### 4.6 `benchmark.py`（对应方案第七节"模型冗余设计"）
- 对每个 run×可用后端：在 val 集算分类 F1/准确率，测单帧推理耗时（ms，多次取均值），测显存/内存占用（MB）。
- 汇总成对比表输出 `runs/benchmark.md` + `runs/benchmark.csv`，作为"选哪个模型 + 哪个分辨率打包进主进程"的依据。

## 5. 数据流

```
make_dummy_dataset.py ──→ dataset_cls/{train,val}/{normal,fault}/  (+ patches/)
copy_paste_aug.py     ──→ 扩充 dataset_cls/train/fault/
                              │
   experiments.yaml ──→ train.py ──→ runs/<name>/weights/best.pt  (×4)
                              │
                    export.py ──→ best.onnx / best_openvino/ / best.engine  (×可用后端)
                              │
                   benchmark.py ──→ runs/benchmark.{md,csv}（F1 / ms / MB 对比）
```

## 6. 错误处理
- 缺依赖（openvino/tensorrt）→ 跳过该后端 + 明确告警，继续其余。
- 无 CUDA → device 回退 CPU；TensorRT 导出自动跳过。
- 数据集为空/路径不存在 → 明确报错并提示先跑 `make_dummy_dataset.py` 或放入真实数据。
- `experiments.yaml` 缺字段 → 校验并给出可读错误。

## 7. 验证（可运行性）
端到端冒烟：`make_dummy_dataset.py` → `copy_paste_aug.py` → `train.py`（1–2 epoch）→ `export.py` → `benchmark.py` 全部跑完不报错，即证明框架可用。真实数据到位后替换 `dataset_cls/` 内容即可正式训练，无需改代码。
