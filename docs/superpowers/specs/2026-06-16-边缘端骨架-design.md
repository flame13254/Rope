# 边缘端骨架设计（Phase 2）

> 智能排绳故障检测与数字孪生系统 —— 边缘工控机运行时代码框架
> 日期：2026-06-16　状态：已确认设计，待评审 spec

## 1. 背景与范围

承接 Phase 1（训练侧）。本阶段交付边缘工控机上的**运行时骨架**：AI 推理主进程 + 视频旁路转发进程 + 部署模板。

当前**无真实摄像头/RTSP 流、无正式训练模型**，故骨架要能在"合成视频源 + MockClassifier + 日志发布"下端到端跑通，真实环境到位后改配置即可切换为 RTSP + 导出模型 + 真实 MQTT broker。

**关键洞察**：Ultralytics `YOLO(path)` 可直接加载 `.pt/.onnx/.engine/openvino` 并推理，"后端可切换"天然落地，推理包装很薄。

**本 spec 覆盖**：AI 主进程、FFmpeg 转发进程、systemd/NSSM 部署模板。
**不覆盖**：动态 ROI 锚点算法实现（仅留接口）、云端 MediaMTX/MQTT Broker 部署、上位机 LabVIEW。

## 2. 已确认的关键决策

| 决策点 | 结论 |
|---|---|
| 范围 | 两进程（AI 主进程 + FFmpeg 转发）+ systemd/NSSM 部署模板 |
| 动态 ROI | 配置固定矩形 ROI + 预留 `ROIProvider` 接口，动态锚点版后续接入 |
| 视频源 | 可插拔 `VideoSource`：合成/文件源现在跑通，配置切换真实 RTSP |
| 推理 | `Classifier` 接口：`MockClassifier`（无需模型）/ `YoloClassifier`（YOLO 加载 pt/onnx/engine/openvino） |
| 发布 | `StatusPublisher`：`MqttPublisher`(paho, QoS1) / `LogPublisher`(可关闭 MQTT, dry-run) |
| 代码结构 | 配置驱动 + 分模块包 `rope_edge/`，薄入口脚本（与 Phase 1 一致） |

## 4. 各组件职责（单一职责、可独立测试）

### 4.1 `rope_edge/config.py`
- 读 `edge.yaml`，校验必需字段，返回结构化配置（dict 或简单对象）。
- 复用 Phase 1 的 `resolve_device("auto")` 思路解析推理设备。

### 4.2 `rope_edge/source.py`
- `VideoSource` 抽象：`frames()` 生成器逐帧产出 `numpy` 图像；`fps` 属性。
- `SyntheticSource`：交替产出"正常平行纹/故障交叉纹"帧（复用 Phase 1 dummy 生成思路），供无摄像头时跑通。
- `FileSource`：`cv2.VideoCapture(path)` 读本地视频文件。
- `RtspSource`：`cv2.VideoCapture(rtsp_url)`，断开自动重连重试。
- `make_source(cfg)` 工厂按 `source.type` 选择。

### 4.3 `rope_edge/roi.py`
- `ROIProvider` 抽象：`crop(frame) -> roi_image`。
- `FixedROI(x, y, w, h)`：按配置矩形裁剪；越界自动夹取到画面内。
- 预留：未来 `DynamicROI`（法兰盘锚点）实现同接口即可替换，pipeline 不改。

### 4.4 `rope_edge/classifier.py`
- `Classifier` 抽象：`predict(roi_image) -> (status:int 0|1, confidence:float 0~1)`。
- `MockClassifier`：按可配置规则（如交替/固定/按帧序）返回 status，无需模型文件，供骨架验证。
- `YoloClassifier`：`YOLO(model_path)` 加载任意后端格式；取 top1，类名为 `fault` → status=1，置信度取该类概率。
- `make_classifier(cfg)` 工厂按 `model.type`(mock|yolo) 选择。

### 4.5 `rope_edge/smoothing.py`（方案"时序平滑"）
- `SlidingWindow(window_sec, ratio, conf_thresh)`：维护时间窗口内的 (timestamp, status, conf) 记录。
- `update(status, conf, now) -> smoothed_status`：窗口内达标帧(status=1 且 conf≥阈值)占比 ≥ ratio 时输出 1，否则 0。
- 纯逻辑，无 IO，TDD 重点。

### 4.6 `rope_edge/publisher.py`（方案"数据通讯协议"）
- `build_payload(camera_id, status, confidence, now) -> dict`：
  `{timestamp: ISO8601+时区, camera_id, status, ai_confidence}`（无 fault_boxes）。
- `StatusPublisher` 抽象：`publish(payload)`。
- `MqttPublisher(host, port, topic, qos)`：paho-mqtt 发布；连接失败/断开 → 告警 + 本地缓存（内存队列），重连后补发。
- `LogPublisher`：打印 JSON（MQTT 关闭/dry-run 时用）。
- `make_publisher(cfg)` 工厂按 `mqtt.enabled` 选择。

### 4.7 `rope_edge/pipeline.py`
- 组合 source/roi/classifier/smoothing/publisher。
- **抽帧解耦**：源按 `source.fps`，推理按 `infer_fps`，按比例抽帧（如 25→6 每隔 N 帧推一次）。
- 仅在平滑后 status **发生变化**时 publish（降低消息量）；可配置心跳间隔定期重发当前状态。
- `run(max_frames=None)`：主循环；`max_frames` 供冒烟/测试限步。

### 4.8 `edge_main.py`
- 解析 `--config configs/edge.yaml`、`--max-frames`(冒烟用)，构建组件并 `pipeline.run()`。

### 4.9 `forward.py`
- 由 `forward.sub_url`/`rtmp_url` 构造 FFmpeg 命令：
  `ffmpeg -rtsp_transport tcp -i <sub_url> -c:v copy -an -f flv <rtmp_url>`。
- `--dry-run` 仅打印命令；否则 `subprocess` 启动并在退出时重启（简单守护）。

### 4.10 部署模板 `deploy/`
- `rope-edge.service` / `rope-forward.service`：systemd，`Restart=always`，`WorkingDirectory` 指向项目，启动 `python edge_main.py` / `python forward.py`。
- `nssm-install.ps1`：用 NSSM 把两脚本注册为 Windows 服务（自启 + 自动重启）。

## 5. 数据流

```
VideoSource.frames() ──> 抽帧解耦(source.fps -> infer_fps) ──> FixedROI.crop()
   ──> Classifier.predict() -> (status, conf)
   ──> SlidingWindow.update() -> smoothed_status
   ──> 状态变化/心跳 ──> Publisher.publish( build_payload(...) )  -> MQTT / 日志

(独立进程) forward.py: RtspSubStream --ffmpeg -c:v copy--> 云端 RTMP
```

## 6. edge.yaml 配置示例
```yaml
camera_id: PTZ_01
source: {type: synthetic, path: "", fps: 25}     # type: synthetic|file|rtsp
infer_fps: 6
roi: {type: fixed, x: 320, y: 180, w: 640, h: 480}
model: {type: mock, path: "", device: auto}       # type: mock|yolo
smoothing: {window_sec: 1.5, ratio: 0.8, conf_thresh: 0.5}
mqtt: {enabled: false, host: 127.0.0.1, port: 1883, topic: rig/data/json, qos: 1}
forward: {sub_url: "rtsp://CAM/sub", rtmp_url: "rtmp://CLOUD/live/spooling"}
heartbeat_sec: 5
```

## 7. 错误处理
- 视频源断开 → 重连重试（带退避），日志告警。
- MQTT 断开 → 告警 + 内存缓存待发，重连后补发；`enabled:false` 时走 LogPublisher。
- 模型加载失败 → 明确报错并退出（部署期暴露问题）。
- ROI 越界 → 夹取到画面范围。
- 配置缺字段 → 可读错误。

## 8. 验证（可运行性）
- 单元测试（无需摄像头/broker/GPU）：config 校验、FixedROI 裁剪与越界、SlidingWindow 判定、payload 字段与时区、MockClassifier、抽帧采样比例。
- 端到端冒烟：`python edge_main.py --max-frames 50`（synthetic 源 + mock 分类器 + LogPublisher）打印若干 status JSON；`python forward.py --dry-run` 打印正确 ffmpeg 命令。
- 真实部署：edge.yaml 改 source=rtsp、model=yolo+导出模型路径、mqtt.enabled=true 即可，无需改代码。
